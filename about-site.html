About this site

What is this

Computer science is just a bunch of puzzles to figure out, I adjusted it so you can work F/T and complete it as a hobby by gamifying the material a little. I routinely eliminate material to increase the efficiency of the curriculum by hopefully reducing it to entirely problem sets and research. The concept is you can complete all the courses using a mobile device, which is what I do while on lunch breaks or commuting, sitting at a coffee shop or waiting in a line. A mobile device is also what most people in the world own, it's difficult to get access to working laptops/desktops and fill it with virtual environments to do other curriculums in some countries.

Game concept

Another reason for the hand-held device concept is this should be like a regular crossword or sudoku puzzle, or a game where you pull it out, solve some challenge in your free time, and toss it away so you can live your life and not be consumed with too much technology. The reward is you learn the fundamentals of math and computer science faster and maybe are motivated to keep doing it. I find this helps reduce the complexity, you are just thinking about a single problem and how to solve it all day in the back of your mind. When you do solve it, or see the solution after thinking enough so the solution is meaningful, you end up learning multiple things all the same time especially with competitive programming and math problems.

What do professors want you to learn

Transfer skills. Topic A is seemingly unrelated to topic B but they actually are related. Traditional curriculums struggle to achieve this as per Prof K's papers on teaching.

Content choice

One bit of advice: it is important to view knowledge as sort of a semantic tree — make sure you understand the fundamental principles, ie the trunk and big branches, before you get into the leaves/details or there is nothing for them to hang on to. - Elon Musk

CompSci

Learn the foundations of computation itself, the trunk and PL theory big branches:

  • CS19 Accelerated Intro to CS and the book Programming and Programming Languages was chosen for having very low complexity and ability to complete on a mobile device. It emphasizes property based testing and design by example. The prof of the course has papers about teaching it: what students ask in office hours, their success at assignments, and how they do in future courses.
  • Robert Harpers PFPL book teaches the foundations of programming languages.
  • Ryan O'Donnell's Theoretical CS Toolkit course goes with the book The Nature of Computation that explains the foundations of computation.
  • Competitive Programming chosen for practice. You can write text source on a tablet or phone and upload it to the online judge servers for free which compiles and tests them as many times as you want. Some of the languages are amenable to templating so often you are just writing the body of the function.

Math

Learn the foundations pure math trunk and some of the big branches of applied math:

  • Tao's book Analysis I and Wildberger's lectures reduce complexity of foundations.
  • The Putnam seminar by Prof Loh and the book Putnam and Beyond because competition problems teach you multiple things at the same time
  • Gilbert Strang's CSE book and MIT lectures to see the common applied framework for using math in all sciences, in a way it's the foundations of applied math

AI

Jump into the leaves. Google Colab can be used with a phone, so since we can practice writing AI algorithms on GPUs through Colab for free may as well also learn optimization of these algorithms. There are numerous AI competitions, most will accept notebook entries you can upload from colab.

Software used to build this

This site is built using org-mode which is a major mode for emacs meaning a collection of behaviours that are automatically run whenever you open a .org file, a text file with some markup. It looks like this: 

#+STARTUP: showall
#+TITLE: About this site
#+options: num:nil html-postamble:nil toc:nil
#+HTML_HEAD: <style> body {background-color: #fafad2} </style>

* Big heading

Some text

** Smaller heading

#+BEGIN_QUOTE
Some text
#+END_QUOTE

The documentation for org html export I find highly confusing, to learn it I had to read the lisp in ox-html.el in emacs source and some other org lisp files in order to find all the features and make my own. You can use HTML5 tags such as for video if you turn on html5-fancy but I don't allowing the user to use their preferred software to look at videos.

I can edit the text on a phone then by a simple script ssh into a box to run: 

emacs myfile.org --batch -f org-html-export-to-html --kill

Which then is upload via version control, now with a minor mode to use persistent div id's instead of org-mode's randomly generated ones, so diffs and direct links are useful. The css background color choice was from a paper in readability, warmer colors are easier for us to read and their compliments at the other end of the color spectrum mix better.

Virtual Private Server

The box that runs emacs is a guix system VPS which is trivial to deploy you essentially edit some s-exp text which are settings inside lists, and it is a single declaration file how exactly you want your remote system set up. Invoking guix deploy installs the remote VM for you on digital ocean or elsewhere. Hacking around your own VPS is possible because the init system, packager, package builds, system tests, and more are all written in a scheme variant which permits any other language you want to compile to their Guile scheme VM bytecode so you don't have to use their preferred language to manipulate your system you can use whatever language you want.

To learn to manipulate your guix system see this book (or pdf) by MIT, they reduce the complexity of learning what an operating system is to the essential details by studying a variant of the V6 original unix system. It is meant to be read while looking at the code.

Guix-HPC is used in high performance computing by INRIA researchers.

Guix-Jupyter notebooks

Guix-Jupyter is self-contained notebooks that describe their execution environment then can be deployed in a completely reproducible way, meaning you can freely install any software you want instead of being limited to what colab offers and run your own VM hosted notebooks to do the AI curriculum on a mobile device if you wanted.

Emacs

You don't have to use notebooks either, if you own a laptop/desktop emacs can be split screen to run an interpreter or compiler beside your code to immediately see output. Pyret, Julia, Common Lisp, Python, it can all be run in emacs. To learn emacs try Xah-Lee's tutorial he reduced the complexity of every other tutorial for emacs like what to do when you randomly come upon some elisp.el file in the wild and wish to use it.
