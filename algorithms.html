What is this

Here we learn the analysis, design and optimization of algorithms by primarily working through the book Competitive Programming 4 (the first 4 chapters) while practicing what we learned with the Kattis Problem Archive so producing working code for hundreds of problems and submitting them for tests of correctness and time complexity to a free online judge. The book is $20 and worth buying, or you can get a previous version from library genesis and just follow along as we investigate algorithms.

The goal here is not just to pass some technical interview at bigtech (you will anyway) but also be able to attempt to solve some of the ICPC archive problems, many which require creative thinking like doing a complete search of the problem domain just to understand what it's asking, manipulating data through multiple modified standard library data structures like manipulating some of the built in libraries to do what they weren't intended to do in order to produce a result for a level 8 difficulty problem in 6 seconds CPU time limit.

We'll do a little bit of everything here like investigating the mathematical properties of trees, complexity theory, computational geometry lectures, and some real life optimization of algorithms for hardware not just abstract models they teach you in intro algorithms books. The source of truth will be of course the solutions you submit to the Kattis online judge but we are going well beyond book 1 with advanced algorithms and data structures.

Languages

The book has a public code repository of all the algorithms contained in it, and explains standard libraries for OCaml, Python 3, Java 11 and C++17 but you can use any of these languages. Pick whatever you personally like from the list of Kattis supported languages, there is no reason to only use C++ because we are doing this primarily to learn and practice algorithm design. If you get judged TLE (time limit exceeded) that's then a challenge to hack the language libraries you chose and find a more efficient way, and all of them can be manipulated like Python with it's ref counter memory management.

If you're using a phone, there are some offline interpreters/compilers you can install from whatever app store you use or try free version of repl.it. You submit to Kattis for judging by pasting in the code through a web editor, uploading a source file(s) or write a script to automate this using their API. You get unlimited judging submissions.

Real contests

We can enter live regional ICPC practice on Kattis, CodeForces, ICFP, Google Kickstart, Facebook Hacker Cup, Code Golf, microcorruption or cryptopals type challenges, most of which we have no hope of winning of course but it's practice to pick up more experience. Apply for a job and the interview is another live contest, you win a signing bonus and salary. If you lose who cares find another interview.

Begin

I'll start with reading chapter 1-2 of Functional Data Structures because it has the best definition of analyzing computational models and their complexity, and the competitive programming book assumes we know this already. When we say 'This algorithm's runtime is O(n)' we are talking about a model of computation and we are analyzing that model which can range from a pointer machine in Erik Demaine lectures to being a completely vague model.
